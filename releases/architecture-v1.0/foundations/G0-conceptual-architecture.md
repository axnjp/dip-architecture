## 1. Platform concept

DIP is a Governed Companion Intelligence Operating System. It provides the
operating environment around replaceable foundation models: persistent
knowledge, governance, workflows, review, memory, orchestration, strategy,
explainability, and auditability.

The platform is intended to preserve long-term architectural capability while
keeping the first implementation narrow enough to be practical. The governing
rule is: **reduce the initial implementation surface, not the governing
architecture.**

Evidence: `recovered_continuity`, `normalized_current`.

## 2. Governing hierarchy

The platform separates:

Foundation Model → AI Operating Profile → Domain Package → Workspace →
Private Strategy

The corresponding platform ownership hierarchy is:

DIP Core → Domain Package → Workspace → Private Strategy

The foundation model supplies reasoning capability; the AIOP governs model
behavior; the Domain Package supplies domain semantics; the Workspace supplies
operational context; and Private Strategy contains private, user-controlled
strategy and intellectual capital.

Evidence: `recovered_continuity`, `normalized_current`.

## 3. Reasoning layers

- Memory: Historian and Event Ledger.
- Knowledge: Knowledge Graph and Knowledge Nodes.
- Intelligence: Findings, Claims, Recommendations, and the Decision Engine.

The Explanation Framework connects intelligence to human interaction. Memory,
knowledge, and intelligence are related but are not collapsed into one store or
one ungoverned model context.

Evidence: `recovered_continuity`.

## 4. Companion-intelligence principles

DIP is designed as a strategic intelligence partner rather than an automatic
agreement engine. It should preserve evidence, explain reasoning, calibrate
confidence, align to user goals, and keep human authority visible. Review
before mutation is a governing principle. Conversation, the Operations Page,
and other authorized interfaces are interaction surfaces; governed services
remain the authority for review, mutation, and audit.

Evidence: `recovered_continuity`, `normalized_current`.

## 5. First-domain scope

Dynasty fantasy football is the first Domain Package. Trevecca’s Finest is the
first operational Workspace. Master Minds is the second Workspace and the
first portability/startup-draft lifecycle validation. Sleeper is read-only.
The first vertical slice is League State Intelligence. Actual Workspace

## 6. Security and disclosure boundary

The publication model distinguishes Public Reference Architecture, Restricted
Architecture, Private Authority, and Secrets. Public documentation may describe
Private Strategy as an abstract capability but must not reveal whether any real
Private Strategy instance exists. Restricted architecture and private authority
remain separately controlled. Secret values never appear in documentation,
repositories, prompts, logs, fixtures, generated files, or archives.
